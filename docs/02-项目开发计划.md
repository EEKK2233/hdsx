# 项目开发计划

## 1. 目标与原则

开发工作由 Agent 持续执行，以小步提交、契约优先、测试先行和可重复验收为原则。每个任务应同时交付代码、迁移、测试和相关文档，不按人员划分工作。

## 2. 里程碑

| 阶段 | 主要成果 | 完成判定 |
| --- | --- | --- |
| M0 环境与骨架 | 环境审计、FastAPI/Vue 骨架、配置、日志 | `llm_learn` 可导入应用；存活接口和基础测试通过 |
| M1 数据与权限 | MySQL、Alembic、用户、课程、RBAC | 迁移可执行；角色越权用例通过 |
| M2 知识库 | 上传、解析、分块、元数据、混合检索 | 固定教材入库且引用可回查 |
| M3 教学能力 | 备课、答疑、作业、批改与复核 | 完成教师和学生关键流程 |
| M4 学情闭环 | 掌握度、学习路径、周/月报 | 报告指标与数据库一致 |
| M5 工程化 | 异步任务、效果评测、安全和部署 | 回归测试与部署说明完整 |

## 3. 当前完成情况

当前已完成 M0～M4 的可运行 MVP：MySQL 8.4 实库与两次迁移、JWT、课程、文档解析与递归分块、Milvus/关键词双路召回、RRF、BGE、Ollama、备课、作业批改与复核、掌握度、学习路径、答疑、家长授权、报告发布、Vue 核心页面、自动化测试和真实端到端冒烟测试。

后续增强项包括：将初始基线迁移完全改为显式 `op`、班级成员完整页面、SSE 流式输出、Celery/Redis 异步任务、Prompt/Skill 管理页面和更大规模离线评测。

## 5. Agent 单任务流程

```text
[读取 AGENTS.md/44.md/相关代码]
 -> [定义一个可验证目标]
 -> [检查工作区与依赖]
 -> [先补契约或失败测试]
 -> [最小实现]
 -> [格式/静态/单元/集成测试]
 -> [更新迁移和文档]
 -> [报告变更、结果和风险]
```

单个任务禁止同时修改多个无关模块。需要联网、安装依赖、修改 Docker 或执行破坏性数据库操作时必须先申请用户授权。

## 6. 质量门槛

- 所有 Python 文件通过 `compileall`。
- 单元测试全部通过；新增功能必须新增测试。
- 迁移在空库可升级，在测试库可回滚。
- OpenAPI 无重复路径，所有写接口有鉴权。
- 模型输出经过 Schema 校验；无证据问答不得正常编造。
- 提交前检查密钥、绝对路径、学生隐私和大文件未进入 Git。

## 7. 风险管理

| 风险 | 应对 |
| --- | --- |
| MySQL 未安装 | 先审计，获权后安装 8.x，不用 SQLite 替代 |
| Milvus/Docker 权限 | 不重建镜像；检查现有服务并记录错误 |
| 8GB 显存不足 | Ollama 7.6B Q4 单任务调用，限制并发和上下文 |
| 模型 JSON 不稳定 | JSON mode、Pydantic 校验、有限修复、失败转人工 |
| 中文 FULLTEXT 效果不足 | 与向量召回融合，后续可接 ES/OpenSearch |
| AI 评分偏差 | rubric、置信度阈值、教师强制复核和离线评测 |
