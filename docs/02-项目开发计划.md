## 5. Agent 单任务流程

```text
[读取 AGENTS.md/44.md/相关代码]
 -> [定义一个可验证目标]
 -> [检查工作区与依赖]
 -> [先补契约或失败测试]
 -> [最小实现]
 -> [格式/静态/单元/集成测试]
 -> [更新迁移和文档]
 -> [报告变更、结果和风险]
```

单个任务禁止同时修改多个无关模块。需要联网、安装依赖、修改 Docker 或执行破坏性数据库操作时必须先申请用户授权。

## 6. 质量门槛

- 所有 Python 文件通过 `compileall`。
- 单元测试全部通过；新增功能必须新增测试。
- 迁移在空库可升级，在测试库可回滚。
- OpenAPI 无重复路径，所有写接口有鉴权。
- 模型输出经过 Schema 校验；无证据问答不得正常编造。
- 提交前检查密钥、绝对路径、学生隐私和大文件未进入 Git。
