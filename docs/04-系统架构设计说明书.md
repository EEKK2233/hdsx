# 系统架构设计说明书

## 2026-07 教学闭环增补

Vue 使用 `KeepAlive` 保存页面实例，答疑会话同时落入 MySQL，实现切换与重新进入后的双层恢复。教师课堂运营聚合 `qa_messages` 与低分 `grading_results`，修正回答后写入通知；学生读取通知时由幂等规则补充 48 小时内未提交作业提醒。作业材料生成仅接收有权课程的已就绪知识库文件，输出经结构校验后写入题目表。

备课 Agent 输出 Markdown 文本，并按教案、讲稿、PPT 提纲和练习使用不同格式约束；数据库以 JSON 包装保存文本以兼容旧记录。作业材料 Agent 优先使用 Ollama JSON Schema 结构化输出，再经过代码块解析、字段别名归一化和受控文本降级。前端顶栏使用路由元数据计算当前位置，并以 sticky 布局固定账号与退出入口。

备课证据路由先执行 MySQL 整句匹配，未命中时运行 Milvus 向量召回、RRF 与 BGE 重排序，仅接受 `rerank_score > 0` 的候选；禁止退回无主题约束的最近文档块。多文件作业生成在 API 层校验全部文件属于当前课程且为 ready，并在 Service 层按文件均衡抽取文档块后合并上下文。

## 1. 架构目标

系统采用模块化单体，优先保证本地可运行、边界清晰和 Agent 可持续维护。FastAPI 承载业务接口，Worker 作为后续独立进程处理耗时任务。该结构可在需要时按知识库、批改和报告模块拆分服务。

## 2. 分层架构

```text
[Vue 3 Web]
     |
[FastAPI API：校验、鉴权、序列化]
     |
[Domain Service：业务规则、事务、状态机]
     |
[Repository/ORM] ---- [Agent Runtime/Tools/Skills]
     |                          |
 [MySQL]             [RAG/Ollama/Reranker/Milvus]
```

API 不直接操作 ORM 或拼 Prompt；Service 不依赖前端；Agent 通过白名单工具读取业务数据，不能直接获取数据库连接。

## 3. 代码结构

- `backend/app/api`：REST 路由、Schema 和鉴权依赖。
- `backend/app/core`：配置、安全、异常。
- `backend/app/db`：SQLAlchemy Base、Engine 和 Session。
- `backend/app/modules`：领域 ORM；后续按 identity/course/knowledge 等拆包。
- `backend/app/rag`：分块、检索、回答类型和知识服务。
- `backend/app/integrations`：Ollama、Milvus、重排序等外部适配。
- `backend/app/services`：文档解析、备课和批改编排。
- `frontend/src`：路由、状态、API 客户端和业务页面。
- `database`、`scripts`、`docs`、`storage`：数据库初始化、运维、文档和文件数据。

## 4. RAG 架构

### 4.1 入库

文件校验 → TXT/MD/PDF/DOCX 解析 → 清洗 → 递归分块（650/100）→ 必需元数据 → embeddinggemma → Milvus → MySQL FULLTEXT → 抽样验证。

当前代码已实现解析、分块、元数据、Ollama Embedding、Milvus 写入和向量召回、MySQL 检索、RRF 融合与 BGE 重排，并已连接真实服务完成验证。为避免 8GB 显存与 Ollama 7.6B 竞争，BGE 默认在 CPU 运行。

### 4.2 查询

权限和课程过滤 → 查询改写 → 向量/关键词双路召回 → 基于 chunk_id/content_hash 去重 → RRF 融合 → BGE 重排 → 上下文构建 → Qwen 生成 → 引用检查 → 返回或转教师。

## 5. Agent 架构

业务 Agent 包括备课、批改、答疑、学情和报告。统一结果语义为 success、needs_review、insufficient_evidence、failed，并包含 data、citations、confidence、warnings 和 trace_id。

工具只提供受控能力，例如 `search_course_knowledge`、`get_assignment_rubric`、`get_student_mastery_summary`。任何写操作经 Service 权限、状态和幂等校验。Skill 固化稳定工作流，Prompt 使用版本号和 JSON Schema。

## 6. 关键业务时序

### 答疑

```text
Student -> API: question
API -> Auth: course permission
API -> RAG: search(course_id, question)
RAG -> MySQL/Milvus: retrieve
RAG -> Reranker: rerank
RAG -> Ollama: grounded prompt
Ollama -> API: answer
API -> MySQL: save message/citations/trace
API -> Student: answer or needs_teacher
```

### 批改

```text
Teacher -> API: grade submission
API -> RuleEngine: objective questions
API -> GradingAgent: subjective answer + rubric
GradingAgent -> Ollama: structured grading
API -> MySQL: save suggestion and evidence
Teacher -> API: final review
API -> MySQL: final score + audit + mastery event
```

## 7. 安全架构

JWT、PBKDF2 密码、RBAC、资源归属共同保护接口。上传文件使用安全文件名和大小限制。文档内容作为不可信数据，不能覆盖系统 Prompt。模型输出需要 JSON/范围校验。CORS 仅允许本地前端地址，生产部署必须改为真实域名。

## 8. 可用性与扩展

- 健康接口区分进程存活和依赖就绪。
- 外部依赖错误转为结构化 503。
- 耗时调用设置 120 秒超时，后续通过 Celery/Redis 异步化。
- MySQL、Milvus、Ollama 均通过适配层，可替换部署地址。
- 文件存储适配层后续可由本地目录切换 MinIO。

## 9. 部署拓扑

开发环境在 Windows 运行 FastAPI、Vue 和 Ollama；MySQL 使用独立 `edu-mysql` 容器并绑定本机 3306，Milvus 使用已有 Docker。Linux 部署建议由 Nginx 提供 HTTPS 和静态文件，后端用 Uvicorn/Gunicorn，MySQL 和 Milvus 使用持久卷，模型服务部署在带 GPU 的主机。

当前项目提供 `start_production.ps1`：先执行 Vue 类型检查和生产构建，再由 FastAPI 同源托管 SPA 与 `/api/v1`，单端口 8000、不启用 reload。公网环境需要在其前方增加 HTTPS 反向代理、正式密钥、访问日志、备份与防火墙。
