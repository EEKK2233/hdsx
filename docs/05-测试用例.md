# 测试用例

## 2026-07 新增验收用例

1. 页面切换后课程、输入和对话不丢失；刷新后可恢复答疑历史。
2. 未保存备课超过 30 条时仅删除最久临时记录，已保存记录不受影响。
3. 家长不可读取未绑定学生；教师不可审批或操作非负责课程。
4. 相同内容不同文件名仍被知识库 SHA-256 去重阻止。
5. 作业按指定知识库文件和章节生成四类材料，学生接口不泄露答案。
6. 薄弱画像仅展示授权学生，推荐内容来自对应课程知识库。
7. 教师修正回答后学生产生未读通知，重新进入对话看到修正标记。
8. 48 小时内到期且未提交的作业只生成一条幂等通知。
9. 家长报告按绑定学生切换，文案不包含难懂的内部字段或原始 JSON。
10. 材料生成接受 Markdown JSON 代码块、嵌入文本和顶层数组。
11. 多选答案顺序不影响判分，少选、多选与错选均不能得满分。
12. 判断题在出题端和学生端均显示正确/错误两个互斥选项。
13. 学生提交后同一请求完成批改，结果接口返回标准答案；主观题结合知识库判分。
14. 报告列表字段为字符串时按完整语句渲染，不得逐字换行。

## 1. 测试策略

测试分为单元、API 集成、外部服务集成、前端和端到端五层。自动化测试默认使用独立 MySQL 测试库；不得以 SQLite 替代 MySQL 行为。Ollama/Milvus 在普通单元测试中使用 mock，在标记的真实集成测试中连接本地服务。

## 8. 非功能测试

- 50 个并发普通查询时记录 P95 延迟和错误率。
- 100MB 非法上传必须在进入模型前拒绝。
- SQL 注入字符串不得改变查询结构。
- 路径 `../../secret` 不得写出 storage。
- 日志扫描不得出现密码、JWT 和 `.env` 内容。
- MySQL 重启后连接池可恢复；Ollama 超时后请求可控失败。

## 9. 端到端验收

教师登录并创建课程，上传教材；检索原文并生成教案；创建含客观和主观题的作业并发布；学生登录提交；教师触发批改并复核；查看学生掌握度；生成周报草稿并审核。全过程不得手工修改数据库。

新增产品功能验证脚本：`python scripts/verify_product_features.py`。脚本已在真实 MySQL、Milvus、Ollama 环境验证批量上传、内容去重、文档清单、学生创建课程 403、教师/学生作业列表以及标准答案隔离。

账号与课程权限验证脚本：`python scripts/verify_account_course_flow.py`。真实验证注册、资料修改、改密、审批前 403、审批后 200、多负责人、CSV 导入和学生课程可见范围。
