# 用户使用手册

## 2026-07 新功能入口

- 教师：在“智能备课”保存或下载历史；在“作业中心”选择知识库文件生成章节材料；在“课堂运营”查看高频问题并修正 AI 回答。
- 学生：在“学情分析”查看薄弱知识画像与推荐内容；在“通知中心”查看截止提醒和教师修正消息。
- 家长：注册时填写一个或多个已有学生用户名，登录后在“学习报告”顶部切换孩子；只能看到对应孩子已发布的报告。
- 所有角色：切换页面不会清空当前操作状态；课堂答疑左侧可恢复最近对话，通知点击后会标记已读并跳转到相关功能。
- 教师在作业详情中可点击每道题后的“修改题目”；已有学生提交后系统会锁定题目，避免改变批改依据。
- 单选、多选和判断题可直接点选标准答案；简答与论述题允许留空，由 AI 结合课程知识库补充，之后仍可修改。
- 学生点击“提交并立即批改”后会直接看到建议得分、批改反馈与标准答案；主观题仍以教师最终复核为准。

## 1. 系统准备

管理员需要 Docker Desktop、已有 Milvus 服务和 Ollama。项目 MySQL 使用 `edu-mysql` 容器，持久化目录为 `D:\Docker\mysql`。Python 必须使用 `llm_learn`。确认 Ollama 已有 `qwen2.5:latest` 和 `embeddinggemma:latest`。

执行环境检查：

```powershell
conda activate llm_learn
cd D:\class\Season4_5\hdsx-d\Code1
python scripts\check_environment.py
```

## 2. 初始化与启动

1. 启动 Docker Desktop，确认 `edu-mysql`、`milvus-standalone`、`milvus-etcd`、`milvus-minio` 正常；首次自行部署 MySQL 时再执行 `database/init_mysql.sql`。
2. 在 `backend` 目录运行 `alembic upgrade head`。
3. 可运行 `python scripts/seed_demo.py` 创建演示数据。
4. 从 `Code1` 启动后端：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_backend.ps1
```

5. 安装前端依赖并启动：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_frontend.ps1
```

后端手动命令必须包含 `--reload-dir backend`，否则 Uvicorn 会递归扫描 `frontend/node_modules`：

```powershell
uvicorn app.main:app --reload --reload-dir backend --app-dir backend
```

浏览器打开 `http://127.0.0.1:5173`。API 文档位于 `http://127.0.0.1:8000/docs`。

项目内已提供便携 Node.js 和 pnpm，普通终端中没有全局 `node/npm/pnpm` 不影响启动。可以使用 `.\bin\node.cmd --version` 和 `.\bin\pnpm.cmd --version` 验证。请勿执行全局 pnpm 安装。

## 3. 首次管理员

若不使用演示脚本，可在 Swagger 中调用 `POST /api/v1/auth/bootstrap` 创建首个管理员。只允许空系统调用一次。随后通过登录接口获取令牌。

演示脚本账号为 admin、teacher、student、parent，统一初始密码 `Demo123!`。只用于本地演示，正式使用必须立即修改。

登录页提供“注册账号”。注册时可以选择学生、教师或家长身份，填写用户名、姓名、可选邮箱和至少 8 位密码；家长必须绑定至少一个已存在的学生用户名。管理员账号仍不能通过公开注册创建。登录后所有角色可进入“账号设置”修改姓名、邮箱和密码。

## 4. 教师操作

### 4.1 创建课程

登录教师账号，进入“课程”，填写课程名称、学科、学段和描述。创建者自动成为第一位负责人。在“成员与负责人”中可添加其他教师为共同负责人、审批学生申请，并查看已加入学生。

批量导入使用 CSV，示例：

```csv
username,display_name,email,password
20260001,张三,zhangsan@example.com,
20260002,李四,lisi@example.com,CustomPass123!
```

未填写每行密码时使用页面统一初始密码。导入后应通知学生首次登录修改密码。

### 4.2 上传教材

进入“知识库”，选择课程和文档分类，一次选择一个或多个 TXT、MD、PDF、DOCX 文件（最多 20 个）。页面逐个显示成功、重复或失败结果。系统根据内容 SHA-256 判断重复，改文件名不能绕过。下方文件清单显示当前课程已有资料和分块状态。

### 4.3 智能备课

进入“智能备课”，输入课程 ID、章节、资源类型、授课对象和课时。系统先检索资料，再调用 Qwen。若显示知识库证据不足，应先上传该章节教材，而不是反复生成。

### 4.4 作业与批改

进入“作业中心”可查看作业卡片、课程、状态、总分、题目数和提交数。点击“创建作业”建立草稿，进入详情后逐题添加并发布。学生提交后，右侧提交队列显示学生姓名和状态；打开提交后启动 AI 辅助批改，再逐题确认最终分数和评语。

主观题必须填写 rubric。AI 分数只是建议，教师需要核对标准答案、评分点和证据。

### 4.5 学情与报告

“学情分析”按学生 ID 查询掌握度，并可生成个性化学习路径。报告页面填写学生、课程和周期，生成结果默认为待审核；教师审核发布后才可作为正式家校通知。

## 5. 学生操作

学生界面不会显示课程创建、文件上传、智能备课、报告生成等教师功能。学生在课程页可发现课程并申请加入；负责人批准前不能读取该课程知识库、作业或答疑。批准后课程进入“已加入课程”。作业中心按课程分组，课堂答疑使用课程名称下拉框；标准答案不会在学生接口或页面中返回。

```json
{"answers":[{"question_id":1,"answer":"A"},{"question_id":2,"answer":"解释内容"}]}
```

学生只能查看本人的学情信息，不能修改 AI 或教师评分。

## 6. 家长操作

家长只能查看已建立亲子授权关系且经教师审核发布的报告。管理员通过家长学生授权接口建立关系，家长通过 `/api/v1/parent/reports` 查看报告。

## 7. 常见问题

### 后端返回 MySQL 连接失败

检查 MySQL 是否启动、端口是否为 3306、数据库和应用账号是否已创建、`.env` 密码是否一致。系统不会改用 SQLite。

### Ollama 调用失败

访问 `http://localhost:11434/api/tags`，确认服务运行且模型名称与 `.env` 相同。8GB 显存环境避免同时运行多个生成任务。

### Milvus 连接失败

检查现有 Docker 服务和 19530 端口。不要重新下载镜像或创建新的 yml；记录容器错误后再处理。

### PDF/DOCX 无法解析

确认不是扫描图片型 PDF、文件未加密且内容非空。扫描件需要后续 OCR 支持。

### 回答没有引用

说明当前关键词未命中或文档尚未完整建立索引。先在知识库页面检索验证，再调整问题或补充教材。

## 8. 停止与维护

使用终端 Ctrl+C 停止前后端。数据库升级前先备份 MySQL 和 storage。不要手工删除 Milvus 向量；应通过后续受控重建索引接口保持 MySQL 与 Milvus 一致。

单端口运行使用 `start_production.ps1`，浏览器访问 `http://127.0.0.1:8000`。公网部署前必须配置 HTTPS、正式随机 `SECRET_KEY`、强数据库密码、防火墙和定期备份。
